# ADR 0006 - i18n setup (progressive migration)

**Status:** Accepted
**Date:** 2026-05-13

## Context

App is currently pt-BR only. Hardcoded strings spread across ~50 components. Future expansion to en requires translation infrastructure.

## Decision

Setup `i18next` + `react-i18next`. Provide pt-BR + en locales. Migrate strings progressively as components are touched, not in bulk.

## Implementation

- Locales in `src/renderer/i18n/locales/{lang}.json` (namespaced: common, alerts, errorBoundary, etc.)
- Bootstrap in `src/renderer/i18n/index.ts`, imported once in `main.tsx`
- Components use `const { t } = useTranslation()` then `{t('namespace.key')}`
- Default lang = pt-BR. No language switcher yet (add when 2nd lang has full coverage).

## Rationale

- Bulk migration of 50+ components = high risk, low value while still single-language
- Setting up infra now means new code uses `t()` from day 1
- Progressive migration touches each component once (during normal work) without dedicated refactor

## Trigger to finish migration

If a second language is needed for production, do full sweep then.
